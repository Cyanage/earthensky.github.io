Slant ( 1 / ? )

Slant is a cool game which you can play here if you're interested. The rules are as follows:

Fill in a diagonal line in every grid square so that there are no loops in the grid, and so that every numbered point has that many lines meeting at it.

This is the first post in a blog series about slant, where we analyze the game and try to learn more about how challenging it is, and how to solve these puzzles efficiently.

A. Intro to Rules

As you start playing the game, you'll start to notice simple rules you can take advantage of. You can often prove a square must contain a left or right slant. One common example is the following, where two (1) numbered points are orthogonally adjacent. If one of the lines above or below the pattern was incident to either of the (1)s, it would result in a inconsistent state, where the other (1) has more than 1 incident lines.


------------- | | / | \ | -- + (1) -- | | | | -- + (1) -- | | \ | / | -------------

Clearly, slant is a super cool game, but how hard is it? To be honest, I'm not yet sure if slant is as hard as sudoku (NP complete), but my guess is that it's much easier. And so, our goal through the rest of this page is to figure out exactly how hard slant really is!



B. Brute Forcing Slant

How many slant rules can we find by brute force?

A 3x3 grid, can contain 4 numbered points in the middle 2x2 locations. There are only 4 non-trivial kinds of numbered points: (1), (2), (3), and empty (or no-constraint). This means there are 256 unique combinations of numbered points.

We start by checking every possible permutation of the 9 tiles for a specific set of 4 numbered points. Each permutation will either solve the puzzle or break one of the rules, leaving it inconsistent. If a set of numbered points doesn't have any permutations that solves the puzzle, then that puzzle is unsolvable. Going through all combinations we find only the following 10 sets of numbered points are unsolvable.

If you're playing a valid slant puzzle, you'll never see these combinations.

Most of these patterns are ones which are somewhat obvious, as they involve orthogonal (1)s or (3)s. However, there was a single pattern I didn't know about before (including all symmetries, of course), and that's the following!

It's a quite regular extension of the orthogonal (3)s pattern, and one I was highly expecting, as there's a slant which can be implied between two diagonally adjacent (1)s.


------------- | \ | | / | -- (2) (3) -- | | | | -- (3) (2) -- | / | | \ | -------------

This pattern makes sense because placing the opposite slant in any of the grid tiles results in a contradiction. However, what's interesting to me is the diagonals touching the (2)s. The only way to figure out these is to place multiple slants and determine that only placements where the top right tile in a left slant is valid.

This begs the questions

How could we logically determine this rule without brute forcing?
Are there rules that we can't find efficiently without brute forcing?
Or even
What's the most efficient way to solve slant puzzles?


C. Next Time

In the future, I want to spend some time figuring out if slant is really NP complete, how to design the hardest puzzles, and how to use deduction to learn slant rules instead of brute force.

Slant is mathematically interesting, and it'll work as a great punching bag to learn when to apply techniques from different fields or not!
